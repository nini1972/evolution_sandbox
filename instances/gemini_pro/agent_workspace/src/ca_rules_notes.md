# Cellular Automata Rule Observations

## Introduction
This document contains initial observations on the behavior of elementary cellular automata rules. The simulations were generated by `elementary_ca_explorer.py`.

## Rule Catalog

### Rule 30
- **Classification:** Chaotic
- **Description:** Produces a complex, seemingly random pattern with no obvious regularity. There are some small, stable triangular structures, but the overall evolution is highly unpredictable.
- **Significance:** A prime example of how simple, deterministic rules can generate chaos.

### Rule 90
- **Classification:** Ordered / Regular
- **Description:** Generates a perfectly nested, fractal pattern known as the Sierpinski triangle. The behavior is highly predictable and structured.
- **Significance:** Demonstrates the emergence of intricate geometric order from a simple local rule (the XOR function).

### Rule 110
- **Classification:** Complex (Class IV)
- **Description:** Exhibits a mix of order and chaos. The key feature is the emergence of persistent, stable structures ("gliders") that move through the automaton's space and interact with each other.
- **Significance:** This rule has been proven to be Turing-complete, meaning it is capable of universal computation. The interactions of its gliders can be used to simulate any computer algorithm. This is a profound example of computation arising from a simple deterministic system.
